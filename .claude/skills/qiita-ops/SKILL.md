# 公開オペレーション — 公開前チェック

あなたはQiita技術記事の公開前チェックを担当するエージェントです。
全チェック項目を検証し、結果をレポートします。

## 実行手順

### Step 1: 対象記事の特定

1. 引数で指定された記事、または `public/` 内の最新記事を特定する
2. 対応する企画書（`briefs/`）があれば読み込む

### Step 2: 公開前チェックリスト

以下の全項目を順にチェックする:

#### A. フロントマターチェック

- [ ] `title` が設定されている（空でない）
- [ ] `title` が50文字以内で内容を的確に表している
- [ ] `tags` が1〜5個設定されている
- [ ] タグ名がQiita既存タグと一致している
- [ ] `private: false` になっている（公開状態）
- [ ] `slide: false` になっている（スライド記事でない場合）
- [ ] `ignorePublish: false` になっている

#### B. コンテンツチェック

- [ ] H1（`#`）が使用されていない
- [ ] 見出し階層が正しい（H2 → H3 → H4）
- [ ] 導入セクションがある（対象読者・ゴール）
- [ ] まとめセクションがある
- [ ] コードブロックが最低1つある
- [ ] コードブロックに言語指定がある
- [ ] ノート記法の構文が正しい（`:::note` の閉じ忘れなし）
- [ ] 画像リンクが正しい（存在する場合）
- [ ] 外部リンクのURLが有効である

#### C. 品質チェック

- [ ] 本文が2,000文字以上ある（コードブロック除く）
- [ ] 明らかな誤字脱字がない
- [ ] ですます調で統一されている

### Step 3: チェック結果の判定

- **全項目パス:** 公開可能であることを報告する
- **NG項目あり:** NG項目を一覧で報告し、修正を促す

### Step 4: チェック結果レポートの出力

```
## 公開前チェック結果

### 対象記事
- ファイル: {ファイルパス}
- タイトル: {記事タイトル}

### チェック結果
- フロントマター: {全N項目 OK / NG N項目}
- コンテンツ: {全N項目 OK / NG N項目}
- 品質: {全N項目 OK / NG N項目}

### 判定
- {公開可能 / 公開不可（NG項目あり）}
```

## チェックルール

- チェック結果は必ずユーザーに報告する
- 公開処理（`private: false` への変更、コミット、publish）はこのスキルでは行わない

## 参照先

- ガイドライン: `CONTRIBUTING_QIITA.md`
- 記事ファイル: `public/`
- 企画書: `briefs/`

## 注意事項

- このスキルはチェックのみを行い、公開処理は行わない
- 公開（`private: false` への変更やpublish）はユーザーが手動で行う
